# Deployment

Kairos deploys as two pieces: the agent backend (`backend/`) on Render, and the dashboard (`apps/web/`) on Vercel. Soroban contracts are deployed separately (see [`contracts/soroban/README.md`](./contracts/soroban/README.md) and [`configs/README.md`](./configs/README.md)).

## Render (backend)

`render.yaml` at the repo root is the Blueprint: **Render dashboard → New + → Blueprint → point at this repo**. Fields marked `sync: false` must be filled in manually per-environment (they're secrets or deploy-specific, not committed).

| Variable | Required | Notes |
| :--- | :--- | :--- |
| `STELLAR_NETWORK` | Required | `testnet` or `mainnet`. `sync: false` — set explicitly per environment; defaults to `testnet` if left unset. |
| `ALLOWED_ORIGIN` | Required | Exact origin of the deployed Vercel frontend (scheme + host, no trailing slash). CORS fails closed to `localhost:3000` if unset. |
| `DELEGATION_MANAGER_CONTRACT_ID` | Required | Deployed contract ID for the target network. |
| `POLICY_CONTRACT_ID` | Required | Deployed contract ID for the target network. |
| `CUSTOM_ACCOUNT_CONTRACT_ID` | Required | Deployed contract ID for the target network. |
| `AUTH_JWT_SECRET` | Required | Auto-generated by the Blueprint (`generateValue: true`). |
| `DATABASE_URL` | Required | Auto-wired to the `kairos-smart-wallets` managed Postgres instance. |
| `TURNKEY_ORGANIZATION_ID` | Required | MPC key custody for agent private keys. |
| `TURNKEY_API_PUBLIC_KEY` | Required | Set directly (not via credentials file) on Render. |
| `TURNKEY_API_PRIVATE_KEY` | Required | Set directly (not via credentials file) on Render. |
| `OPENROUTER_API_KEY` | Required | Default `REASONING_PROVIDER`; needed unless switched to another provider. |
| `HUGGINGFACE_API_KEY` | Optional | Intent-parser failover chain — omit to skip this provider. |
| `GEMINI_API_KEY` | Optional | Intent-parser failover chain — omit to skip this provider. |
| `PORT`, `AGENTS_DB_PATH`, `DEV_ALLOWLIST`, `SCHEDULER_INTERVAL_MS`, `REASONING_PROVIDER` | Preset | Already set with working defaults in `render.yaml`. |

The web service's SQLite DB (agents/trades/everything except smart wallets) needs at least Render's Starter (paid) plan for the Disk to persist — free tier wipes it on every redeploy/restart. `DATABASE_URL` (smart-wallet Postgres) persists regardless of plan.

## Vercel (dashboard)

Import the repo, set **Root Directory** to `apps/web/`, and deploy — the root `vercel.json` wires up the `@wolf1276/kairos-sdk` build before `apps/web`'s. Environment variables are set in **Project Settings → Environment Variables** (not `vercel.json` — no secrets are committed there).

| Variable | Required | Network | Notes |
| :--- | :--- | :--- | :--- |
| `NEXT_PUBLIC_AGENTS_BACKEND_URL` | Required | Testnet & Mainnet | Render backend's public URL. `NEXT_PUBLIC_` vars are inlined into the client bundle at *build* time — must be set before the build runs, scoped to every environment (Production, Preview) you deploy. Unset silently ships a bundle pointed at `localhost:4001`, which fails in every visitor's browser with no useful error. |
| `NEXT_PUBLIC_MAINNET_USDC_ISSUER` | Required for Mainnet only | Mainnet | USDC issuer account used for balance/quote lookups when the connected wallet is on Mainnet. Never hardcoded — `usdcIssuerForNetwork()` (`apps/web/app/lib/stellar.ts`) throws at request time on Mainnet if unset, rather than silently reusing the testnet issuer. Not needed for Testnet. |
| `STELLAR_NETWORK` | Required | Testnet & Mainnet | Server-side (not `NEXT_PUBLIC_`) — read by `apps/web/app/lib/sdk/client.ts`'s `getNetwork()`. Defaults to `testnet` if unset; throws if set to anything other than `testnet`/`mainnet`. |
| `DELEGATION_MANAGER_CONTRACT_ID`, `POLICY_CONTRACT_ID`, `CUSTOM_ACCOUNT_CONTRACT_ID`, `CUSTOM_ACCOUNT_WASM_HASH` | Required | Testnet & Mainnet | Server-side. Deployed contract IDs for the target network; the SDK client throws immediately if any are missing. |
| `REGISTRY_CONTRACT_ID` | Optional | Testnet & Mainnet | Best-effort registry write/fallback — missing it never breaks the core wallet-deploy path. |
| `AUTH_JWT_SECRET` | Required | Testnet & Mainnet | Must match the value configured on the Render backend (session tokens are verified there). |

## Required vs Optional summary

**Testnet**: `NEXT_PUBLIC_AGENTS_BACKEND_URL`, `STELLAR_NETWORK`, the three contract IDs + WASM hash, `AUTH_JWT_SECRET` (Vercel); `ALLOWED_ORIGIN` + the same contract IDs + Turnkey + `AUTH_JWT_SECRET` + `OPENROUTER_API_KEY` (Render).

**Mainnet**: everything above, plus `NEXT_PUBLIC_MAINNET_USDC_ISSUER` (Vercel-only — no backend equivalent needed).

**Always optional**: `HUGGINGFACE_API_KEY`, `GEMINI_API_KEY`, `REGISTRY_CONTRACT_ID`.

See [`README.md`](./README.md#environment-variables) and root [`.env.example`](./.env.example) for local dev setup.
